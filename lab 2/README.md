## **Теоретическая часть**
В работе рассматриваются основные абстрактные типы данных:

### **1. List (Динамический массив)**
- **Структура**: последовательность элементов, хранящихся в непрерывном блоке памяти
- **Принцип доступа**: быстрый доступ по индексу (O(1))
- **Преимущества**:
  - Константное время доступа к элементам по индексу
  - Эффективное использование кэша процессора
  - Минимальные накладные расходы памяти
- **Недостатки**:
  - Медленная вставка/удаление в начале (O(n))
  - Требуется перераспределение памяти при расширении
- **Оптимальное применение**: частый доступ к элементам по индексу, редкие вставки в начало

### **2. LinkedList (Связный список)**
- **Структура**: цепочка узлов, каждый содержит данные и ссылку на следующий элемент
- **Принцип доступа**: последовательный доступ по ссылкам
- **Преимущества**:
  - Быстрая вставка/удаление в любом месте (O(1) при известном узле)
  - Динамическое расширение без копирования данных
  - Не требуется предварительное выделение памяти
- **Недостатки**:
  - Медленный доступ по индексу (O(n))
  - Дополнительная память для хранения ссылок
  - Неэффективное использование кэша процессора
- **Оптимальное применение**: частые вставки/удаления, особенно в начале или середине списка

### **3. Stack (Стек)**
- **Принцип**: LIFO (Last In, First Out — последний вошёл, первый вышел)
- **Основные операции**:
  - `push()` — добавление элемента на вершину стека
  - `pop()` — удаление элемента с вершины стека
  - `peek()` — просмотр вершины стека без удаления
- **Временная сложность**: O(1) для всех основных операций
- **Типичные применения**:
  - Проверка сбалансированности скобок
  - Обработка выражений (постфиксная запись)
  - Отмена операций (undo)
  - Обход графов в глубину

### **4. Queue (Очередь)**
- **Принцип**: FIFO (First In, First Out — первый вошёл, первый вышел)
- **Основные операции**:
  - `enqueue()` — добавление элемента в конец очереди
  - `dequeue()` — удаление элемента из начала очереди
  - `front()` — просмотр первого элемента без удаления
- **Временная сложность**: O(1) для всех основных операций (при правильной реализации)
- **Типичные применения**:
  - Обработка задач в порядке поступления
  - Буферизация данных
  - Моделирование реальных процессов
  - Обход графов в ширину

### **5. Deque (Двусторонняя очередь)**
- **Принцип**: поддержка операций с обоих концов
- **Основные операции**:
  - `appendleft()`, `append()` — добавление слева/справа
  - `popleft()`, `pop()` — удаление слева/справа
- **Временная сложность**: O(1) для всех основных операций
- **Преимущества**:
  - Гибкость в работе с обоими концами
  - Эффективные операции вставки/удаления
- **Типичные применения**:
  - Планировщики задач
  - Скользящие окна в алгоритмах
  - Палиндромы и симметричные структуры
