## Теоретическая часть

### Основные понятия

**Куча (Heap)** — специализированная древовидная структура данных, удовлетворяющая свойству кучи. Является полным бинарным деревом (все уровни заполнены, кроме последнего, который заполняется слева направо).

**Свойство кучи:**
- **Min-Heap**: Значение в любом узле меньше или равно значениям его потомков. Корень — минимальный элемент.
- **Max-Heap**: Значение в любом узле больше или равно значениям его потомков. Корень — максимальный элемент.

**Реализация на массиве:**
Для узла с индексом `i`:
- Родитель: `(i-1)//2`
- Левый потомок: `2*i + 1`
- Правый потомок: `2*i + 2`

**Основные операции:**
- **Вставка (insert)**: O(log n) — элемент добавляется в конец и "всплывает" (sift-up)
- **Извлечение корня (extract)**: O(log n) — корень извлекается, последний элемент становится корнем и "погружается" (sift-down)
- **Построение кучи (build_heap)**: O(n) — преобразование произвольного массива в кучу (алгоритм Флойда)

**Применение:**
- Сортировка кучей (HeapSort)
- Реализация приоритетной очереди
- Алгоритм Дейкстры
