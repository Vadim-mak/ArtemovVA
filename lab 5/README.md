## Теоретическая часть
Хеш-функция — это функция, которая преобразует произвольные данные (например, строку) в числовое значение фиксированной длины (хеш-код).

Основные требования к хеш-функции:
- Детерминированность (одинаковый вход → одинаковый выход)
- Быстрое вычисление
- Равномерное распределение значений

Хеш-таблица — структура данных, реализующая ассоциативный массив. Она состоит из массива ячеек (бакетов), каждая из которых может хранить один или несколько элементов.

Коллизия — ситуация, когда два разных ключа имеют одинаковый хеш-код. В данной работе реализован метод цепочек (chaining), при котором элементы с одинаковым хешом хранятся в виде списка в одной ячейке.

Временная сложность операций:
- Вставка: O(1) в среднем случае
- Поиск: O(1) в среднем случае
- Удаление: O(1) в среднем случае
- В худшем случае: O(n)

## Ответы на контрольные вопросы
1. Каким требованиям должна удовлетворять "хорошая" хеш-функция?
   - Детерминированность
   - Быстрое вычисление
   - Равномерное распределение
   - Минимизация коллизий

2. Что такое коллизия в хеш-таблице?
   Коллизия — ситуация, когда два разных ключа имеют одинаковое значение хеш-функции.

3. Почему операции в хеш-таблице в среднем выполняются за O(1)?
   Вычисление хеша и доступ к массиву по индексу — операции O(1). В среднем длина цепочек невелика.

4. Что такое коэффициент заполнения?
   Отношение количества элементов к общему числу ячеек. При высоком коэффициенте возрастает вероятность коллизий.

5. Какую структуру использует метод цепочек?
   Связный список (в реализации — Python list) для хранения элементов с одинаковым хешем.
